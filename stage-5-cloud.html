Практический разбор

Миграция с VPS в AWS без потери инженерной дисциплины

Cloud не отменяет production-baseline. Те же требования к секретам, логам, backup, healthcheck и rollback должны быть сохранены, но теперь выражены через managed-сервисы и IAM.

ОбластьРешение для стартаЧто важно контролировать
ComputeEC2 + systemd или Docker ComposeAMI, patching, instance role, autoscaling
DatabaseRDS PostgreSQLbackup window, maintenance window, encryption
FilesS3 private bucketbucket policy, lifecycle, presigned URLs
NetworkVPC with public/private subnetsroute tables, NAT, security groups
SecretsSSM Parameter Store или Secrets Managerrotation, access audit, least privilege
Практика

Сначала повторите текущую VPS-архитектуру на EC2, затем постепенно заменяйте части managed-сервисами: S3 для файлов, RDS для PostgreSQL, CloudWatch/Prometheus для наблюдаемости.
