Практический разбор

Минимальный production pipeline

CI/CD должен не просто запускать команды, а снижать риск поставки. Разделите pipeline на независимые jobs и не деплойте артефакт, который не прошёл quality gates.

ШагЧто проверяетТипичный fail-fast критерий
lintСтиль, форматирование, базовые ошибкиruff/mypy завершается с ошибкой
testsБизнес-логика и API contractsЛюбой unit/integration test упал
buildDockerfile и runtime-зависимостиImage не собирается или healthcheck отсутствует
publishImmutable image tag в registryНет digest или tag перезаписывается
deployДоставка на VPS и smoke check/health не отвечает после rollout
deploy.yml
name: deploy
on:
  push:
    branches: [main]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: ruff check . && pytest
  build:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - run: docker build -t registry.example.com/app:${{ github.sha }} .
      - run: docker push registry.example.com/app:${{ github.sha }}
