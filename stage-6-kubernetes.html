Практический разбор

Что значит production-ready workload

Kubernetes-проект считается зрелым не когда Pod запустился, а когда rollout управляем, ресурсы ограничены, конфигурация отделена от image, а Git отражает желаемое состояние cluster.

deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: fastapi
spec:
  replicas: 3
  strategy:
    type: RollingUpdate
  template:
    spec:
      containers:
        - name: api
          image: registry.example.com/app:sha
          readinessProbe:
            httpGet: { path: /health, port: 8000 }
          resources:
            requests: { cpu: "100m", memory: "128Mi" }
            limits: { cpu: "500m", memory: "512Mi" }

Probes

Readiness отвечает за трафик, liveness — за восстановление зависшего процесса, startup probe помогает долгому старту без ложных рестартов.

Limits

Requests нужны scheduler, limits защищают cluster от runaway-процессов. Начинайте с измерений, а не случайных чисел.

Helm values

Все отличия окружений должны быть в values: replica count, image tag, ingress host, resource profile и feature flags.

GitOps

ArgoCD/Flux сравнивает Git и cluster, показывает drift и позволяет откатываться через revert коммита.
